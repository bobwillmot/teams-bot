# Project Documentation

This folder is the canonical home for project documentation that should live in the repository.

## Structure

- `agent-transcripts/`: dated logs of significant agent requests and outcomes
- `adr/`: architecture decision records for durable technical decisions

## Naming Convention

- Keep overview files as `README.md` where that conventional name is strongest.
- Use lowercase hyphenated Markdown file names for other documentation files.
- Name transcript files as `yyyy-mm-dd.md`.
- Name ADR files as `0001-short-title.md`, `0002-short-title.md`, and so on.

## Notes

Use the folders in this directory for ongoing project tracking rather than creating parallel documentation locations elsewhere in the repository.
